# WebStyle：赛博霓虹科技风规范

## 适用范围

本文档用于约束 `web/public` 前端的视觉与交互风格，当前覆盖：

- `web/public/dashboard.html`
- `web/public/dashboard.css`
- `web/public/dashboard.js`

不涉及 `web/dist`、React/Vite 前端、后端 API、WebSocket 协议或场景 JSON 格式。

## 风格定位

整体方向是“赛博霓虹指挥舱”：深色背景、青蓝/紫粉/电红高亮、HUD 网格、玻璃面板、实时链路动画和终端式日志。

页面应该像一个实时 Agent 网络态势面板，而不是传统后台表格或纸面仪表盘。视觉可以炫酷，但核心仍然是可读、可操作、可监控。

## 布局规范

主布局保持稳定比例：

- 外层：`canvas-panel : side-panel = 3 : 1`
- 左侧内部：`canvas stage : scene panel = 1 : 1`
- 窗口横向放大时，canvas 与 side panel 按比例共同扩展
- 窗口横向缩小时，canvas 与 side panel 共同收缩，不允许 canvas 固定宽度导致 side panel 被挤压折叠
- 小屏宽度不足时允许上下堆叠，但不能遮挡日志、控制区或场景面板

场景面板不使用悬浮窗形式，必须内嵌在左侧区域下半部分，并带有 HUD 标题栏。

## 色彩系统

统一使用 CSS 变量管理颜色，不在 HTML 中写主要行内样式。

推荐语义：

| 变量 | 用途 |
| --- | --- |
| `--bg` | 页面主背景 |
| `--panel` | 面板玻璃底色 |
| `--panel-strong` | 强调面板底色 |
| `--border` | 细线边框 |
| `--text` | 主文本 |
| `--muted` | 次级文本 |
| `--cyan` | 主科技高亮、链路、按钮 |
| `--blue` | 冷色辅助高亮 |
| `--magenta` | Agent、警示、强调光 |
| `--green` | 成功、运行、在线 |
| `--yellow` | 等待、注意、广播 |
| `--orange` | 警告、冲突、任务焦点 |
| `--red` | 错误、失败、异常 |
| `--purple` | 深层状态、控制节点 |

避免回到米色、纸张、单一灰蓝或传统后台浅色风格。

## 面板规范

面板采用半透明玻璃质感：

- 深色半透明背景
- 细线霓虹边框
- 柔和外发光
- 顶部可使用细扫描光条
- hover 状态允许增强边框和阴影

不要把页面大区块做成多层嵌套卡片。卡片只用于日志项、重复条目、工具容器或必要的局部信息块。

## Canvas 视觉规范

Canvas 是页面主视觉，应承担“实时网络态势图”的角色。

背景：

- 深色径向或线性渐变
- HUD 网格
- 轻量星点/噪声
- 扫描线或扫描光带
- 支持 `prefers-reduced-motion` 降级

Agent 节点：

- 发光核心
- 外环描边
- 状态脉冲
- hover 时增强外环和信息提示
- 选中或活跃状态应比普通状态更亮，但不能影响名称可读性

关系线：

- 使用霓虹链路效果
- 可使用虚线、光晕、轻微流动感
- 真实通信但无拓扑边时，允许绘制临时淡色通信线

通信轨迹：

- 每条消息生成一条报文轨迹
- 轨迹包含移动头部、拖尾、目标端点脉冲
- 同一条边多条消息使用轻微垂直偏移，避免完全重叠
- 高频消息必须裁剪旧事件，避免性能下降

## 交互规范

Canvas 交互以鼠标为核心：

- 滚轮以鼠标指针位置为锚点缩放
- 左键拖空白区域平移画布
- 左键拖 Agent 时移动 Agent，不触发画布平移
- 双击空白区域恢复自动 fit 视图
- tooltip、命中检测、拖拽、连线、动画必须走同一套世界坐标与屏幕坐标转换

交互反馈需要明确：

- 可平移时显示 grab
- 平移中显示 grabbing
- Agent 可拖动时使用 move/grab 类反馈
- 按钮、选择器、折叠控件要有霓虹 hover/active 反馈

## 日志规范

侧边栏日志采用终端/数据流风格。

日志展示要求：

- 时间、来源、类型、消息主体层级清晰
- 错误、系统、Agent、通信、Docker 流量等分类具有不同状态色
- 长文本必须换行或裁剪，不能撑破面板
- 新日志可以闪入，但不能频繁触发布局抖动
- 日志内容以可读为先，不为了装饰牺牲排查效率

全局日志格式变化时，前端日志页面应优先兼容统一的 `global.jsonl` 事件结构，再向旧字段做降级读取。

## 文案规范

HUD 标签可以使用短英文，例如：

- `AGENT NETWORK`
- `SCENE FEED`
- `LIVE BUS`
- `GLOBAL LOG`

功能性文本优先使用中文，保持简短直接。

避免在页面中加入大段说明、操作教程或键盘快捷键介绍。交互应该通过控件形态、图标、tooltip 和状态反馈自然表达。

## 动效规范

默认动效强度可以偏高，但必须可控：

- 背景扫描
- Agent 呼吸
- 报文尾迹
- 日志闪入
- 按钮能量反馈

性能约束：

- 不新增第三方依赖
- 不为每条日志创建昂贵动画
- 通信轨迹、粒子、脉冲必须设置数量上限
- 使用 `performance.now()` 统一动画时间
- 避免混用 `Date.now()` 与 `performance.now()` 造成动画异常

降级策略：

- 当系统开启 `prefers-reduced-motion: reduce` 时，关闭扫描、闪烁、强脉冲等持续动画
- 保留静态科技风，包括深色主题、边框、状态色和基础布局

## 代码组织规范

HTML：

- 保留语义结构
- 移除主要行内样式
- 控件使用稳定 class
- 不改变后端 API 或页面入口

CSS：

- 颜色、阴影、透明度优先走 CSS 变量
- 布局使用 grid/flex，避免固定宽度挤压
- 响应式规则集中维护
- reduced motion 单独收口

JS：

- Canvas 绘制逻辑保持模块化函数
- 世界坐标与屏幕坐标转换集中维护
- 通信事件生成与绘制解耦
- DOM 更新要兼容缺失节点，避免控制台硬错误
- 动画循环中避免不必要的 DOM 查询和对象暴增

## Do / Don't

| Do | Don't |
| --- | --- |
| 使用深色玻璃面板和霓虹边框 | 回到纸面、浅色后台或米色卡片风 |
| 保持 3:1 与 1:1 布局比例 | 让 canvas 固定宽度挤压 side panel |
| 用 CSS 变量统一视觉系统 | 在 HTML 里堆行内颜色和阴影 |
| 让 Agent、链路、报文都有状态表达 | 只做静态圆点和普通线条 |
| 高频日志和消息做数量裁剪 | 无限堆积动画事件 |
| 支持 reduced motion | 强制所有用户播放高强度动效 |
| 优先保证日志可读性 | 为了炫酷牺牲排查效率 |

## 验收清单

修改 `web/public` 前端后，应至少确认：

- 页面整体是深色赛博霓虹主题
- 左右主布局保持 3:1
- 左侧 canvas 与场景面板保持上下 1:1
- 窗口横向缩小时 side panel 不被 canvas 挤没
- Canvas 缩放、平移、拖 Agent、双击 reset 正常
- Agent 节点、关系线、通信轨迹方向正确且可读
- 日志长文本不溢出、不遮挡、不撑破布局
- 高频消息下页面不卡顿
- `prefers-reduced-motion` 下动画能降级
- 浏览器控制台没有明显运行时错误
