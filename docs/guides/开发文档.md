# AgentNetwork 开发文档

Agent 通信相关修改必须同时遵守 [ADR-024-CommManager统一A2A通信与禁止广播.md](../adr/ADR-024-CommManager统一A2A通信与禁止广播.md)；任务下发、状态机与回调修改必须同步更新 [ADR-025-Agent任务下发与A2A回调.md](../adr/ADR-025-Agent任务下发与A2A回调.md)。不得恢复广播、`DirectBus` 或中心 relay。

## 1. 本地启动

```bash
docker compose config
docker compose up -d --build
```

默认服务：

- `srv`：控制面与 Web，宿主机端口 8000；
- `ag-o1`：OpenCLAW 运行槽位；
- `ag-c1`：Claude Code 运行槽位。

当前 compose 不启动 `bus`。需要零 token 演示时显式设置 `MOCK_LLM=1`。

## 2. 代码模块

```text
services/
  server.py                 FastAPI 控制面和 Dashboard
  agent_server.py           Agent 容器 HTTP 运行时
  message_bus.py            已废弃的 410 兼容桩

agent_network/
  api/
    managed_simulations.py  仿真生命周期与任务 HTTP 适配
    simulations.py          底层场景执行、抓包和网络配置
    agents.py               Agent 查询接口
    logs.py                 日志查询、写入、文件管理
    packets.py              PCAP 查询、分析、审计、打包
    system.py               系统信息与配置
  agent_management.py       AgentRegistry、ContainerRuntime、按事件执行
  scene_management/        统一剧本定义、存储与业务管理
    scene_def.py            场景与 Agent 数据结构
    models.py               Skill、Tool、Task、摘要与校验结果模型
    validator.py            agentnetwork-scene.v1 严格多问题校验
    scene_storage.py        单剧本存储、解析与归档
    scene_manager.py        剧本单项及批量业务管理
  simulation_management/   统一仿真管理模块
    models.py               仿真状态、运行配置、事件与控制信号
    event_scheduler.py      阻塞式事件队列、稳定排序与并发派发
    resource_allocator.py   逐 Agent 资源计划和容量校验
    simulation_manager.py   配置、启动、停止、强停和持续时间控制
    state.py                仿真控制面共享状态
  adapters/
    base.py                 AgentContext / AgentRunResult
    claude_code.py          Claude Agent SDK Adapter
    openclaw.py             OpenCLAW SDK Adapter
    direct_llm.py           显式直接模型能力
  native_capabilities.py    Claude/OpenClaw 原生能力规范名、策略与安全校验
  native_audit.py           原生 Tool 授权、结果和子 Agent 生命周期审计
  comm_management/         统一 A2A 通信管理模块
    comm_manager.py        Agent Card、权限、顺序发送与 Task 回执
    network_emulation.py   延迟、抖动、丢包、带宽和 Linux tc 配置
  capture_management/      统一抓包管理模块
    http_adapter.py        Agent 抓包 HTTP 适配
    packet_store.py        PCAP 注册、解析、查询和分析
  task_management/         SQLite Task 生命周期与 A2A Push 回调

  log_management/          统一日志管理模块
    log_manager.py          严格日志 schema、索引、持久化

    log_batch_install.py    批量能力安装适配
    llm_metrics.py          LLM 调用元数据日志

```

旧顶层通信、抓包、网络仿真、Skill 和状态入口已删除。通信与网络条件使用 `agent_network.comm_management`；抓包控制和 PCAP 查询使用 `agent_network.capture_management`；仿真共享状态使用 `agent_network.simulation_management.state`；Skill 源读取只通过 `mcp_server --skill-source-mode` 暴露。

## 3. 主要 API

### 仿真

- `POST /api/simulations/configure`
- `POST /api/simulations/{simulation_id}/start`
- `POST /api/simulations/{simulation_id}/stop`
- `POST /api/simulations/{simulation_id}/force-stop`
- `GET /api/simulations`
- `GET /api/simulations/{simulation_id}`
- `POST /api/simulations/{simulation_id}/agents/{agent_id}/tasks`
- `POST /api/simulations/{simulation_id}/task-events`
- `GET /api/scenes`
- `GET /api/scenes/{scene_key}`
- `GET /api/scenes/state`
- `GET /api/scenes/{scene_name}`

### Agent 容器内部接口

- `POST /run`
- `GET /.well-known/agent-card.json`
- `POST /a2a/v1/message:send`
- `GET /a2a/v1/tasks` 与 `GET /a2a/v1/tasks/{task_id}`
- `POST /a2a/v1/tasks/{task_id}:cancel`
- `POST /message`（迁移期只读兼容入口）
- `GET /status`
- `GET /inbox`
- `POST /reset`
- `POST /capture/start`
- `POST /capture/stop`
- `GET /capture/status`
- `POST /network/configure`
- `POST /network/clear`
- `POST /internal/native/policy/check`（Bearer Token，OpenClaw Plugin 内部使用）
- `POST /internal/native/audit`（Bearer Token，Tool/子 Agent 终态）
- `GET /internal/native/children/{agent_id}`（Bearer Token，父子树快照）

### 日志与数据包

- `/api/logs/*`：查询、统计、导出、下载、隐藏、删除和 ingest；
- `/api/packets/*`：查询、统计、分析、PCAP 下载、实验 manifest、质量审计、bundle 和跨层关联；
- `/ws`：Dashboard 实时状态、日志和 packet 更新。

## 4. 开发流程

全部修改遵循 [ADR-030-设计驱动与可追溯变更流程.md](../adr/ADR-030-设计驱动与可追溯变更流程.md)。

1. 先阅读 `docs/README.md`、ADR-030、相关设计文档和实际代码。
2. 确认变更属于控制面、Agent runtime、通信、Skill/Tool、日志或网络证据中的哪一层，并确定权威设计记录位置。
3. 修改代码、配置、脚本、测试、部署或文档时，同步记录动机、边界、实现映射和验证证据。
4. 实现时同步更新生产者、消费者、测试和设计文档。
5. 早期阶段不添加无需求兼容代码；旧接口应删除或明确返回已废弃。
6. 不修改 `scenes/` 下指导场景编写的文档，除非任务明确要求。
7. 对日志、通信和网络采集的变更，优先检查是否破坏证据真实性。
8. 完成前检查完整 diff；证据链不完整时不得提交、推送或合并。

## 5. 最低验证集

文档或代码变更完成后至少执行：

```bash
python scripts/check_design_traceability.py
python -m pytest
docker compose config
```

涉及 Claude/OpenClaw 原生能力时还必须执行：

```bash
node --check docker/openclaw-audit-plugin/index.mjs
python -m pytest tests/test_native_capabilities.py tests/test_native_audit.py tests/test_adapter_context_boundary.py tests/test_dashboard_native_events.py
```

涉及后端镜像时再执行对应构建与验收：

```bash
docker compose build ag-c1
docker compose build ag-o1
./scripts/check-openclaw-runtime.sh ag-o1
```

涉及网络仿真或抓包时验证：

- 容器有 `NET_ADMIN`、`NET_RAW`；
- `tc` 和 `tcpdump` 可用；
- launch 后每个 Agent capture 状态为 running；
- stop 后 PCAP 有效且 hash 可验证；
- `/api/packets/quality?session_id=...` 通过。

## 6. 日志变更检查

修改日志时必须同时检查：

- `log_manager.py` schema 与版本；
- 所有 emitter；
- API 查询和 ingest；
- Dashboard 字段读取；
- token usage 聚合；
- experiment audit；
- 原生 Tool 请求/结果和子 Agent 终态完整性；
- Dashboard 对 `policy_check`、`tool_call_requested`、`tool_result_received`、`subagent_lifecycle` 的分类；
- tests；
- `docs/design/日志与实验数据设计.md`。

禁止仅在 reader 中增加兼容分支来掩盖 producer 未迁移。

## 7. 通信与网络变更检查

修改消息或网络路径时必须同时检查：

- `CommManager` 的 exact ID、双向 allowlist 与 A2A Agent Card；
- `agent_directory` 和 `comm_matrix` 下发；
- MCP `send_message`，不提供广播；
- 多目标顺序发送与 A2A inbox 语义；
- 拓扑到 `tc` profile 的转换；
- 控制面抓包过滤；
- PCAP/manifest/quality audit；
- Claude/OpenClaw 原生消息和 OpenClaw A2A 保持关闭，不能绕过 CommManager；
- `docs/design/通信与网络仿真设计.md`。

## 8. 文档验收

- `docs/README.md` 中链接均存在；
- 设计文档不引用已经删除的 `bus` 数据面、旧 logger、旧 Skill 注入或不存在的文件；
- 新设计决策已写入 `docs/adr/`，并登记到 `docs/adr/README.md`；
- 后端原生能力变更已同步 ADR-032、版本锁定、Hook/Plugin、日志 v12、manifest v3 和 Dashboard；
- 每项修改都能追溯到设计记录、实现位置、测试或检查以及实际验证结果；
- “未执行”与“已通过”严格区分，受限验证必须记录原因、风险和后续责任人；
- diff 中不包含未明确要求的 `scenes/` 文档修改。
