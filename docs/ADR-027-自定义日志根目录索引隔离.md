# ADR-027：自定义日志根目录索引隔离

- 状态：已接受
- 日期：2026-07-17
- 关联：ADR-020、ADR-023、统一日志管理设计

## 背景

`LogManager` 在使用非默认日志目录时会创建独立的 `FileManager`。原实现把资源索引写入日志目录的父目录：

```text
<log_dir_parent>/file_registry.json
```

多个互为兄弟目录的日志根因此共享同一索引。隐藏当前日志文件后，列表查询仍可能返回其他日志根中同类型的可见资源，表现为隐藏操作不可见或测试间数据污染。

## 决策

1. 项目默认日志目录继续使用项目级共享 `FileManager` 和既有统一注册表；
2. 每个非默认日志根使用根内独立索引：`<log_dir>/.file_registry.json`；
3. 自定义日志根之间不得读取、修改或列出彼此的资源元数据；
4. 日志隐藏、显示、下载和删除仍统一委托 `FileManager`，不引入第二套可见性状态；
5. 增加兄弟日志根隔离测试，防止索引路径重新退回父目录。

## 结果

- 单个日志文件的可见性查询只受所属日志根索引影响；
- 测试临时目录、嵌入式运行和自定义部署日志目录互不污染；
- 默认生产部署的共享文件注册表契约保持不变。

## 禁止回退

- 不得让多个非默认日志根隐式共享父目录中的注册表；
- 不得通过列表结果后置过滤来掩盖跨根索引污染；
- 不得恢复独立的 `.log_visibility.json` 可见性机制。
