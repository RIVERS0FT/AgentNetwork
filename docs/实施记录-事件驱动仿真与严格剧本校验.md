# 事件驱动仿真与严格剧本校验实施记录

> 状态：已实施。本文是本次架构迁移的防回退记录；与旧文档中的“当前差距”冲突时，以本文和对应 ADR 的已实施合同为准。

## 1. 事件驱动仿真

- `SimulationRuntimeConfig` 显式保存持续时间、Agent 超时、空闲超时、A2A 网络模式和资源分配。
- `SimulationEventScheduler` 按 `simulation_id` 管理事件队列，使用可执行时间和稳定序号排序；无事件时通过条件变量阻塞，不做周期空转。
- 初始任务、Agent 就绪和外部下发任务都转换为 `SimulationEvent`。同一 Agent 单任务串行，不同 Agent 在 `max_parallel_agents` 内并行。
- 调度上下文使用 `simulation_id`、`event_id`、`event_sequence` 和 `event_type`，不再使用固定执行次数或调度循环编号。
- 停止和强制停止会唤醒调度器并取消待处理事件。结束结果记录处理、失败、取消事件数及停止原因。
- 场景状态接口和 Dashboard 展示事件计数；实验清单版本升级为 `agent-traffic-experiment.v2`。

权威实现：

- `agent_network/simulation_management/models.py`
- `agent_network/simulation_management/event_scheduler.py`
- `agent_network/simulation_management/simulation_manager.py`
- `agent_network/api/managed_simulations.py`
- `agent_network/api/simulations.py`

## 2. 严格剧本校验

- 三个核心文件固定为 `meta_and_roles.json`、`instances_and_skills.json`、`network_topology.json`，使用 `agentnetwork-scene.v1` 严格字段合同。
- `SceneValidator` 一次收集全部可定位问题，结果包含 `validated_at`、错误数、警告数和 `issues[]`。
- 校验覆盖 Agent 非空与标识唯一、角色/实例双向对应、后端、Skill 安全路径与入口文件、Tool 注册与重复、任务结构/授权/依赖及循环、拓扑端点/信道/网络值和跨文件引用闭合。
- `SceneStorage` 是唯一单剧本解析与校验入口；上传校验失败时回滚已提升目录并清理临时数据。
- `SceneDefinition` 独立持有 Agent、Skill、Tool、Task 和拓扑定义；列表固定返回 `SceneListItem`，详情固定返回 `SceneSummary`。
- 顶层可运行剧本统一改用 `network_topology.json`，并删除剧本元数据中的旧调度字段。
- 顶层可运行剧本的 Tool、Skill 参数、业务事件和面板字段统一使用 `event_sequence`，不得把场景内部数据重新解释为固定轮次调度。

权威实现：

- `agent_network/scene_management/models.py`
- `agent_network/scene_management/validator.py`
- `agent_network/scene_management/scene_storage.py`
- `agent_network/scene_management/scene_manager.py`

## 3. 防回退要求

1. 不得恢复固定次数或停滞计数调度，也不得在 API、Agent 上下文、Dashboard 或 manifest 中恢复旧调度字段。
2. 不得绕过 `SimulationManager` 建立第二套生命周期状态机或事件队列。
3. 不得绕过 `SceneStorage`/`SceneValidator` 在 API 或编排层复制剧本解析和校验规则。
4. 不得把列表扩展为详情接口，也不得把原始文件、内部资源 ID 或运行配置混入 `SceneSummary`。
5. 修改上述合同必须同步修改 ADR、本文、自动化测试和 manifest/schema 版本。

## 4. 验证基线

- 全部验证必须设置 `MOCK_LLM=1`，不得调用真实模型。
- 自动化测试覆盖事件排序与失败、禁止旧调度合同、严格多问题报告、上传回滚、列表/详情合同和 manifest v2。
- 顶层可运行剧本 `ap_deployment`、`tech_campus`、`telecom_bidding_test` 必须通过完整语义校验。
