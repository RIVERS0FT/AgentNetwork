# ADR-023：LogManager 统一批量管理与会话可见性

- 状态：已接受
- 日期：2026-07-15
- 关联：统一文件管理、日志与实验数据、SceneManager 批量管理

## 背景

项目已有基于 `FileManager` 的统一日志持久化，但批量下载、跨会话批量删除、批量解析、批量可见性和整会话可见性仍不完整。剧本领域已经由 `SceneManager` 提供统一批量编排、逐项隔离和结果汇总，日志领域需要保持同等级别的管理能力。

同时，日志会话是 `application.jsonl`、`network.jsonl` 和 `system.jsonl` 的共同父资源。只管理单文件可见性无法表达“隐藏整个实验日志会话”，也可能导致会话隐藏后新写入文件重新可见。

## 决策

1. 继续使用唯一的 `LogManager`，不新增第二个日志管理单例；
2. 批量业务方法安装到当前 `UnifiedLogManager`；
3. 批量操作统一返回 `LogBatchResult` 和逐项 `LogBatchItemResult`；
4. 支持批量下载、批量删除、批量解析和批量文件可见性；
5. 支持单会话和批量会话可见性；
6. 会话目录可见性是父级门控，文件可见性是子级门控；
7. 默认列表、解析和下载必须同时满足两级可见性；
8. 会话隐藏期间创建的新日志文件继承隐藏状态；
9. 活动日志会话禁止批量删除；
10. 所有物理文件操作继续委托 `FileManager`；
11. 批量下载归档必须注册为受管资源；
12. 批量解析只读原始 JSONL，不修改源文件。

日志领域模块统一收敛到 `agent_network/log_management/`：

- `log_manager.py`：日志 schema、规范化、索引与基础管理；
- `log_batch.py`：批量操作与会话可见性；
- `log_batch_install.py`：向当前 UnifiedLogManager 安装批量能力；
- `llm_metrics.py`：外部模型调用元数据日志；
- `__init__.py`：日志领域公共导入入口。

不得在 `agent_network/` 顶层恢复同名实现文件；内部调用统一从 `agent_network.log_management` 或其子模块导入。

## 结果

- `LogManager` 与 `SceneManager` 具有一致的批量任务、逐项错误隔离和结果汇总能力；
- 会话可见性可以统一控制同一实验的全部日志；
- 单个文件可见性仍可独立调整，但不能绕过隐藏父会话；
- 日志写入期间切换可见性不会影响继续持久化；
- 隐藏和删除保持不同语义；
- API 层不再需要自行拼装日志归档或操作文件注册表。

## 禁止回退

不得重新引入以下设计：

- 第二个日志管理单例或独立日志资源管理器；
- API 直接调用 `FileManager` 批量删除或压缩日志；
- 只隐藏当前已有文件、但让会话后续新日志恢复可见；
- 把隐藏等同于删除；
- 批量任务首项失败后终止全部项目；
- 活动会话批量删除；
- `.log_visibility.json` 独立可见性状态；
- 下载接口暴露服务器物理路径。

修改日志管理职责、批量结果模型或会话可见性语义时，必须同时更新本 ADR、统一日志管理设计、API 和测试。
