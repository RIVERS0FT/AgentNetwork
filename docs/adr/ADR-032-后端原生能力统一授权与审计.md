# ADR-032：后端原生能力统一授权与审计

- 状态：已接受并实现
- 日期：2026-07-21
- 适用范围：Claude Code、OpenClaw、Agent 容器运行时、原生 Tool、原生子 Agent、应用日志、实验清单与 Dashboard
- 关联需求：SR-SIM-05、SR-LOG-06

## 背景

Claude Code 和 OpenClaw 都提供文件、进程、网络、浏览器、子 Agent、消息或自动化等后端原生能力。若直接把这些后端 Tool 名称视为剧本 Tool，既会绕过场景 allowed_tools、MCP 与 CommManager，也无法形成一致的授权、审计和实验复现证据。原生子 Agent 还会引入父子关系、独立 session、并发和终态完整性问题。

## 决定

### 1. 原生能力与场景 Tool 分离

场景 Tool 继续由 tools.py、allowed_tools 和 MCP 管理。后端原生能力使用独立的 native_capabilities 策略，采用后端无关的规范名：

    fs.read  fs.search  fs.write  process.exec
    web.fetch  web.search  browser.control
    agent.spawn  automation.schedule
    device.control  media.generate

agent.message 和 channel.send 只用于表达被禁止的后端能力，不允许配置到 allow。Agent 间消息必须继续通过 MCP send_message 和统一 CommManager 的 A2A 点对点路径发送。

默认 profile 为 audited_read_only：允许 fs.read、fs.search、web.fetch、agent.spawn；写文件、执行进程、浏览器控制、搜索、消息、Cron、设备与媒体生成默认拒绝。未知 Tool、未知策略字段、目录越界、敏感凭证路径、危险命令、私网 URL 和无法安全解析的 URL 一律拒绝。

### 2. Claude Code 强制点

Claude Agent SDK 使用 Claude Code 原生 Tool preset，同时配置真实的 allowed_tools、disallowed_tools、strict_mcp_config=True 和 permission_mode=dontAsk。

- PreToolUse 在执行前调用统一策略并返回 allow/deny；
- PostToolUse、PostToolUseFailure 记录结果、错误、时延和文件修改前后哈希；
- SubagentStart、SubagentStop 记录父子生命周期；
- AgentDefinition(agentnetwork-worker) 限定原生子 Agent 的 Tool 和模型；
- MCP Tool 仍由原有 MCP 权限与日志链负责，Claude Hook 不重复记录；
- SDK session ID 持久化，并仅在策略允许时 resume。

任何 Hook、SDK 或策略初始化失败都不得静默退化为 direct LLM；该规则不依赖环境开关。

### 3. OpenClaw 强制点

OpenClaw Gateway 必须加载 agentnetwork-audit 插件：

- before_tool_call 调用 Agent 容器内 /internal/native/policy/check，审计服务不可用、Token 错误或策略拒绝时 fail-closed；
- after_tool_call 调用 /internal/native/audit 记录终态结果；
- subagent_spawned、subagent_ended 记录父子生命周期；
- subagent_ended 的 outcome 按 ok/error/timeout/killed/reset/deleted 规范化；outcome 缺失时按 error 字段判定 failed，否则记为 completed，避免把自由文本 reason 当成非终态；
- 内部接口使用 NATIVE_AUDIT_TOKEN Bearer Token；未配置 Token 时拒绝请求；
- OpenClaw 原生 Agent-to-Agent Tool 关闭，避免绕过 CommManager；
- Gateway、Plugin 或 SDK 不可用时运行失败，不允许自动 fallback。

### 4. 原生子 Agent 模型

原生子 Agent 是父 Agent 后端内部的受限执行单元，不是新的容器槽位，也不是可绕过通信矩阵的独立平台通信主体：

- 父子共享父 Agent 的容器身份；
- 子 Agent 使用独立 runtime session/run ID；
- 生命周期投影到 AgentRegistry，记录 parent_agent_id、runtime_kind、runtime_session_id、spawn_depth 和状态；
- 默认最多创建 3 个子 Agent、最大深度 1、最多并行 2 个；
- 默认子 Agent 不可继续 spawn；
- Gateway 的绝对并行上限为 32，具体运行仍由每个 Agent 策略的默认并行 2 或显式更小配额强制限制；
- reset 时必须清理审计状态并注销原生子 Agent 投影；
- completed、failed、cancelled、terminated、timeout、killed、reset、deleted 均为终态。

### 5. 审计与实验完整性

application.jsonl 升级为 application.v12，新增或扩展：

- policy_check
- tool_call_requested
- tool_result_received
- subagent_lifecycle

Tool 输入和输出保存 SHA-256，并按策略保存脱敏预览；文件写入保存修改前后哈希。每个允许的原生 Tool 请求必须有结果终态，每个已启动子 Agent 必须有生命周期终态。

Experiment manifest 升级为 agent-traffic-experiment.v3，记录：

- 每个 Agent 的完整原生策略与 SHA-256；
- Claude Code、Claude Agent SDK、OpenClaw、OpenClaw SDK 精确版本；
- 原生子 Agent 树；
- Tool 请求/结果和子 Agent 终态完整性审计。

Dashboard 将上述四类事件归入 Agent/Application 时间线，展示 parent/child、backend、session、capability、policy decision 和状态。

### 6. 版本固定

    Claude Code       2.1.216
    Claude Agent SDK  0.2.124
    OpenClaw          2026.7.1-2
    OpenClaw SDK      2.1.0

镜像构建不得使用 latest 或未校验的本地 wheel 通配符。

## 被放弃方案

1. 把后端原生 Tool 合并进场景 allowed_tools：会混淆剧本能力和运行时能力，无法跨后端稳定授权。
2. 只依赖后端自己的 allowlist：缺少 AgentNetwork 的统一拒绝规则、日志 Schema 和 Manifest 证据。
3. 允许原生消息或 OpenClaw A2A：会绕过 CommManager、通信矩阵和 A2A 审计。
4. 为每个原生子 Agent 创建新容器：成本高，且不符合 Claude/OpenClaw 原生 session 执行模型。
5. 审计服务故障时继续执行：会产生不可审计实验，因此拒绝。

## 迁移、回滚与失败语义

- 旧剧本没有 native_capabilities 时使用默认只读审计策略；direct_llm 始终禁用原生能力。
- audit.required 必须保持 true，输入输出至少保存哈希；持久 session 超过 ttl_seconds 后不得 resume。
- 非法字段或不可授权的原生消息能力在剧本校验阶段失败。
- OpenClaw 内部审计 Token 缺失、错误或接口不可达时 Tool 调用被阻止。
- 预创建和动态创建的 Agent 容器都必须继承 NATIVE_AUDIT_TOKEN 与原生路径边界；Agent 容器无法把原生审计写入控制面时授权链抛错并停止，不能吞错继续执行。
- Compose 不提供已知默认 Token；Token 为空时内部接口返回 401，原生能力保持失败关闭。
- OpenClaw SDK 2.1.0 的异步 factory 必须使用 `await OpenClawClient.connect()` 后再进入 async context；不得恢复旧式未 await 调用。
- Claude Hook 或 OpenClaw Plugin 已允许但未产生结果终态时，实验质量审计失败。
- 子 Agent 缺少终态时，实验质量审计失败。
- 回滚必须整体回滚策略模块、Hook/Plugin、内部接口、日志 v12、manifest v3、Dashboard 和文档；不得只关闭审计而保留原生执行。
